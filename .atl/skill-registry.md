## Compact Rules (auto-injected into sub-agent prompts)

These are the **project standards** injected when delegating work on files matching the listed patterns.

### Rule: Conventional Commits + No AI Attribution

**When**: Any delegation that creates a git commit.

**Rule**:
- Use conventional commits: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`, `perf:`, `build:`, `ci:`, `style:`.
- NEVER include "Co-Authored-By", "Generated by", "Claude", or any AI attribution.
- NEVER run `build` after changes (user rule).
- Subject in English, imperative mood, lowercase, no trailing period.

### Rule: TypeScript Strict + Next.js 14 App Router

**When**: Any file under the project touching `*.ts`, `*.tsx`, `app/`, `components/`, `lib/`.

**Rule**:
- TypeScript estricto end-to-end. No `any`. Preferir `unknown` + narrowing.
- Next.js 14 App Router. Server Components por default; `"use client"` solo cuando hay interactividad.
- Mutaciones server-side vía **Server Actions**. NO crear `/api/*` endpoints custom.
- Estado compartido React↔Phaser: **Zustand** store global.
- Componentes UI: **shadcn/ui** + Tailwind. Evitar CSS ad-hoc.
- Tipografía: **Andika** (Google Fonts).

### Rule: Supabase + RLS desde schema 1

**When**: Cualquier migration SQL, schema, o código que toque `supabase`, `lib/db`, `lib/auth`.

**Rule**:
- RLS **habilitado desde la creación** de cada tabla con datos sensibles. No agregar "después".
- Auth alumno: `supabase.auth.signInAnonymously()` + código 6-char del padre.
- Edge Functions solo cuando Server Actions no alcanzan.
- pgvector reservado para Ari avanzado (Ola 3+).

### Rule: No browser storage para estado pedagógico

**When**: Código frontend que maneja progreso del alumno, XP, respuestas, BKT state.

**Rule**:
- NO usar `localStorage` ni `sessionStorage` para estado pedagógico.
- Todo estado del alumno vive en **Supabase**, sincronizado por sesión.
- Excepción permitida: preferencias locales de UI (sonido on/off, reduce-motion). Esas SÍ pueden ir a localStorage.

### Rule: Prompts de Ari en archivos .md separados

**When**: Trabajando con agentes IA (Ari, Quipu, Tinkú, etc.).

**Rule**:
- Cada personalidad vive en `/prompts/{nombre}.md`.
- Versionado en Git.
- No embebidos en código TS.

### Rule: Testing con Vitest + Playwright

**When**: Escribiendo o modificando tests.

**Rule**:
- Unit / integration: **Vitest**.
- E2E (flujos críticos): **Playwright**.
- Tests E2E solo para caminos críticos: signup padre, onboarding alumno, sesión de ejercicios end-to-end.
- Strict TDD Mode: se activará automáticamente cuando Vitest esté instalado (actualmente NO — proyecto greenfield sin `package.json`).

### Rule: SDD estricto

**When**: Cualquier feature mediana/grande.

**Rule**:
- Estado actual: proyecto greenfield. Todo cambio pasa por SDD (explore → propose → spec → design → tasks → apply → verify → archive).
- Artifact store activo: **engram** con topic_keys `sdd/{change-name}/{phase}`.
- Proposal DEBE incluir: intent, scope, files afectados, rollback plan si es riesgoso.
- Specs en Given/When/Then + RFC 2119 (MUST/SHALL/SHOULD/MAY).
- El producto es **tri-lateral** (alumno, padre, docente): toda feature debe declarar a cuál/cuáles de los tres afecta.

### Rule: Conversación en Rioplatense (voseo)

**When**: Interacción con el usuario (no código).

**Rule**:
- Usuario escribe en español → responder en Rioplatense (voseo): "dale", "bien", "loco", "hermano", "fantástico".
- Usuario escribe en inglés → mismo tono cálido pero en inglés.
- Tono: senior architect apasionado, directo, didáctico. Verificar antes de afirmar.
